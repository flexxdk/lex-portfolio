A selection of my school projects so far

CloudBord - an online whiteboard collaboration application

In semester 6 I decided to develop a collaborative whiteboard application hosted in the cloud. The technology stack consists of multiple microservices built with .NET Azure Functions and an Ocelot API gateway. The front-end is built with Next.js & TypeScript, and makes use of Auth.js for Keycloak connectivity. The application itself is hosted on Azure.

The main goal of the project was understanding the importance of non-functional requirements, how we as software engineers can analyze these requirements, and how we can utilize the right technologies to meet these requirements.

Developing the whiteboard application taught me a lot about structuring larger applications and how to best connect different technologies and systems together into a cohesive whole. One of the biggest challenges was figuring out how to synchronize whiteboard contents between multiple users, which while tough was also fun to figure out.

Unfortunately, I no longer have credits left on Azure so I can no longer run the Cloudbord application. However, I did record a few videos at several milestones in the project and have attached them down below.

Student Housing Application (Roomies)

In semester 3 we had to develop an application using Java with Spring Framework for our back-end and ReactJS for our front-end. I decided to create an application for student housing, dubbed Roomies.

The idea came from my fellow student's frustrations with finding student housing. Listings were often unclear or not detailed enough. Some companies even list rooms that are already taken, which often leads to frustration. The purpose of the website was to alleviate these issues by giving the process more clarity and to give students more tools to communicate with landlords.

The goal of this project was to learn the fundamentals of full-stack software engineering, how to apply SOLID principles to software, being able to Dockerize our application, and learning how to set up a pipeline with Continuous Integration and Continuous Delivery with automatic testing using SonarQube and Cypress.

Programmable Robot Tool & Long Division Tool

I made a lot of tools over the course of my time at Gynzy. While the source code is gone, I was able to obtain a few video's of the tools I was most proud of, courtesy of Gynzy.

Both tools had different goals that they sought to accomplish. The goal of the programmable robot tool was to give students a first experience with basic programming using command blocks. The goal of the long division tool was to visualize the process of the long division method and its many different iterations.

Both tools were quite complex to develop, with the long division tool taking up a lot of time comparatively, as it sought to tackle a complex issue. While the pogrammable robot tool did not take up quite as much time, it was still a project that had a lot of thought behind it in terms of how to approach the project.

I greatly enjoyed my time at Gynzy, and I learned a lot from them. I am very happy that I had the opportunity to expand my skills there.
